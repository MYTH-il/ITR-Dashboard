# ITR Dashboard

A tax return workflow management and tracking system built with FastAPI + React + MongoDB.

## Quick Start — Self-Hosted Deployment

Deploy this app to AWS EC2 + MongoDB Atlas in ~15 minutes:

### Prerequisites (one-time setup)

1. **AWS EC2 instance** — Launch a `t3.micro` Ubuntu instance (free tier)
   - Allocate an Elastic IP and associate it with the instance
   - Security group: allow inbound 22 (your IP only), 80, 443 (open)

2. **MongoDB Atlas** — Create a free M0 cluster
   - Add a database user + password
   - Add this instance's public IP to Network Access
   - Copy the connection string (you'll need it during deployment)

3. **DuckDNS subdomain** — Register free at duckdns.org
   - Point your subdomain to the Elastic IP
   - Let's Encrypt blocks bare IPs and EC2 auto-hostnames — you need a real registered domain

### Deploy

SSH into your instance and run:

```bash
curl -O https://raw.githubusercontent.com/MYTH-il/ITR-Dashboard/main/deploy.sh
chmod +x deploy.sh
./deploy.sh
```

The script handles everything:
- System dependencies (Python, Node, Nginx, Certbot)
- Cloning the repo
- Environment configuration (.env generation with user input)
- Backend setup (Python venv + dependencies)
- Frontend build (React)
- Systemd service + Nginx + HTTPS

When prompted, provide:
- Your DuckDNS subdomain (e.g., `itr-dashboard.duckdns.org`)
- MongoDB Atlas connection string (`MONGO_URL`)
- Admin password

That's it — the app will be running at `https://your-subdomain.duckdns.org`.

## Cost

This deployment is designed to stay lightweight enough for a small EC2 instance
and MongoDB Atlas M0, but it should not be treated as guaranteed `$0/month`.

- AWS free-tier credits and eligible instance offers depend on the AWS account,
  region, current AWS terms, and usage.
- AWS charges separately for public IPv4 addresses, including Elastic IPs and
  public IPv4 addresses attached to running EC2 instances.
- MongoDB Atlas M0 and Let's Encrypt can be used without a paid plan within
  their published limits.

Check AWS Billing/Cost Explorer after deployment and set a billing alert before
using the app in production.

## Development

### Local setup

This project is developed locally on CachyOS/Arch-based Linux and deployed to
Debian/Ubuntu servers. Keep OS-specific setup in development tooling only; the
production deploy script should remain Debian/Ubuntu compatible.

### Local preview

The preview runner starts the FastAPI backend and React frontend together:

```bash
chmod +x scripts/preview.sh
./scripts/preview.sh
```

Preview URLs:

- Frontend: `http://localhost:3000`
- Backend API: `http://localhost:8001/api`
- Default admin login: `admin@taxops.com` / `Admin@123`

The script creates `backend/.env` and `frontend/.env` if they do not already
exist, creates `backend/venv`, installs backend requirements, installs frontend
packages with Yarn, and runs both dev servers.

MongoDB is required before starting the preview. If Docker or Podman Compose is
available, use:

```bash
docker compose -f compose.preview.yml up -d
# or
podman compose -f compose.preview.yml up -d
```

On this CachyOS development machine, MongoDB can also run from the local
repo-local install:

```bash
scripts/mongo-local.sh start
scripts/mongo-local.sh status
```

On CachyOS/Arch, install the main local tools with:

```bash
sudo pacman -S python nodejs yarn
```

On Debian/Ubuntu, install equivalent local tools with:

```bash
sudo apt-get update
sudo apt-get install -y python3 python3-venv python3-pip nodejs
sudo npm install -g yarn
```

Manual setup is still available:

```bash
cd frontend
yarn install
yarn start

# In another terminal
cd backend
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
uvicorn server:app --reload
```

### Production build

```bash
cd frontend
yarn build  # produces frontend/build/ (static files)

cd backend
pip install -r requirements.txt
# Run via systemd service in production
```

**Note:** Always use `yarn`, not `npm`. The `package.json` contains a `resolutions` field that npm doesn't understand.

## Architecture

- **Backend:** FastAPI + Motor (async MongoDB), Uvicorn
- **Frontend:** React + Tailwind + shadcn/ui
- **Database:** MongoDB Atlas (free tier) or self-hosted
- **Server:** Nginx (reverse proxy) + systemd (process manager) + Let's Encrypt (HTTPS)

## Configuration

Environment variables in `.env` files (generated by `deploy.sh`):

### `backend/.env`
```
MONGO_URL=mongodb+srv://user:pass@cluster.mongodb.net/?retryWrites=true&w=majority
DB_NAME=itr_dashboard
JWT_SECRET=<auto-generated>
ADMIN_EMAIL=admin@example.com
ADMIN_PASSWORD=<your-password>
CORS_ORIGINS=https://yourdomain.com
RESEND_API_KEY=<optional>
ESCALATION_CHECK_INTERVAL_MINUTES=30
```

### `frontend/.env`
```
REACT_APP_BACKEND_URL=https://yourdomain.com
```

## Troubleshooting

**Backend won't start**
```bash
sudo journalctl -u itr-backend -n 50 --no-pager
```
Check logs for the actual error (usually MongoDB connection or missing environment variable).

**502 Bad Gateway**
The backend isn't listening on port 8001. Check `systemctl status itr-backend`.

**Can't request HTTPS certificate**
Make sure your DuckDNS domain actually resolves to this instance:
```bash
dig yourdomain.duckdns.org
```

**Yarn vs npm**
This project requires `yarn`, not `npm`. The `package.json` uses yarn's `resolutions` field for dependency pinning, which npm ignores entirely.
